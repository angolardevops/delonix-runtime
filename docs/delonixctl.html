delonixctl cliente PaaS

O cliente REMOTO do Delonix PaaS — homólogo ao kubectl. Produto irmão deste engine, dum repositório diferente (privado): fala só HTTP com a API /v2/* de uma plataforma Delonix PaaS já a correr — nunca toca em containers/VMs directamente (isso é o delonix documentado no resto deste site).

delonixdelonixctl
PapelMotor local de containers/VMs/redeCliente remoto do PaaS
Onde correNa máquina do Delonix EngineDe qualquer máquina
Fala comO kernel directamenteSó HTTP com /v2/*
Cliente remoto do Delonix PaaS (fala só com a API /v2)

Usage: delonixctl [OPTIONS] <COMMAND>

Commands:
  auth        Autenticação de tenant (device-flow) — partilha `~/.delonix/auth.json` com o `delonix auth login` do runtime local
  team        Teams (`/v2/teams`)
  addons      Addons geridos (`/v2/addons`) — postgres/pgvector/redis/kafka/minio/mongodb/qdrant
  app         Apps por preset (`/v2/apps`) — nextcloud/onlyoffice/mattermost/odoo/collab/coturn
  org         Organizações (`/v2/orgs`)
  odoo        Apps Odoo (`/v2/odoo*`) — superfície própria (módulos, ops, ficheiros). Os outros presets (nextcloud/onlyoffice/mattermost) usam `delonixctl app create --preset <nome>`, não têm grupo próprio
  apply       Aplica um manifesto (`/v2/apply`, admin — pode criar/atualizar QUALQUER Kind)
  delete      Remove o(s) recurso(s) do manifesto (`/v2/delete`, admin)
  completion  Imprime o script de autocompletion do shell (bash/zsh/fish/...)
  version     Versão do cliente E do servidor ligado (padrão `kubectl version`) — substitui o `--version`/`-V` gerado pelo clap
  help        Print this message or the help of the given subcommand(s)

Options:
      --api <API>      URL da API (ex.: https://host:9443). Também `DELONIX_API`
      --insecure       Aceita certificados TLS self-signed (a API vem assim por omissão). Também `DELONIX_API_INSECURE=1`
      --token <TOKEN>  Token Bearer explícito (sobrepõe `DELONIX_TOKEN`/ficheiro de login)
      --json           Imprime a resposta em JSON em vez de tabela legível, onde aplicável
  -h, --help           Print help

Guia completo (auth, todos os grupos, troubleshooting) no repo do PaaS: docs/user/DELONIXCTL.md.

delonixctl auth

Device-flow (RFC 8628) contra o portal do teu operador — nunca pede password directamente.

Autenticação de tenant (device-flow) — partilha `~/.delonix/auth.json` com o `delonix auth login` do runtime local

Usage: delonixctl auth [OPTIONS] <COMMAND>

Commands:
  login   Autentica-se no portal (device-flow) ou aceita `--token` directo
  status  Mostra se há credenciais guardadas (sem revelar o token)
  logout  Remove as credenciais guardadas
  help    Print this message or the help of the given subcommand(s)

Options:
      --api <API>      URL da API (ex.: https://host:9443). Também `DELONIX_API`
      --insecure       Aceita certificados TLS self-signed (a API vem assim por omissão). Também `DELONIX_API_INSECURE=1`
      --token <TOKEN>  Token Bearer explícito (sobrepõe `DELONIX_TOKEN`/ficheiro de login)
      --json           Imprime a resposta em JSON em vez de tabela legível, onde aplicável
  -h, --help           Print help

Exemplos

Autenticar (abre um código no browser)
delonixctl auth login --portal https://portal.o-teu-operador.example
Confirmar credenciais guardadas
delonixctl auth status
Sem device-flow, token já em mão (CI/automação)
delonixctl auth login --token "o-teu-token"

delonixctl team

Teams (`/v2/teams`)

      --json           Imprime a resposta em JSON em vez de tabela legível, onde aplicável
  -h, --help           Print help

Exemplos

Listar
delonixctl team ls
(sem teams)
Criar/substituir membros

delonixctl app

Apps por preset — nextcloud/onlyoffice/mattermost/odoo/collab/coturn.

Apps por preset (`/v2/apps`) — nextcloud/onlyoffice/mattermost/odoo/collab/coturn

Usage: delonixctl app [OPTIONS] <COMMAND>

Commands:
  ls       Lista as apps (visíveis ao token actual)
  presets  Catálogo de presets disponíveis
  create   Cria uma app por preset (assíncrono — devolve um job)
  job      Estado/log de um job de provisionamento
  help     Print this message or the help of the given subcommand(s)

Options:
      --api <API>      URL da API (ex.: https://host:9443). Também `DELONIX_API`
      --insecure       Aceita certificados TLS self-signed (a API vem assim por omissão). Também `DELONIX_API_INSECURE=1`
      --token <TOKEN>  Token Bearer explícito (sobrepõe `DELONIX_TOKEN`/ficheiro de login)
      --json           Imprime a resposta em JSON em vez de tabela legível, onde aplicável
  -h, --help           Print help

Exemplos

Catálogo
delonixctl app presets
ID           NOME           DESCRIÇÃO
nextcloud    Nextcloud      Ficheiros + colaboração; Postgres gerido, HTTPS.
onlyoffice   OnlyOffice     Document Server; edição de documentos (JWT).
mattermost   Mattermost     Chat de equipa + Calls (vídeo/voz); Postgres gerido.
odoo         Odoo           ERP/CRM; Postgres gerido, filestore, ingress.
collab       Suite colab    Nextcloud + OnlyOffice + Mattermost integrados.
coturn       coturn (TURN)  Relay TURN/STUN para media WebRTC (RTC).
Criar (assíncrono — devolve um job)
delonixctl app create a-minha-app --preset nextcloud --domain files.example.com
Bloquear até terminar
delonixctl app create outra --preset odoo --wait

delonixctl odoo

Único preset com grupo próprio (módulos, operações, ficheiros) — os outros usam só `app create --preset`.

Apps Odoo (`/v2/odoo*`) — superfície própria (módulos, ops, ficheiros). Os outros presets (nextcloud/onlyoffice/mattermost) usam `delonixctl app create --preset <nome>`, não têm grupo próprio

Usage: delonixctl odoo [OPTIONS] <COMMAND>

Commands:
  images  Tags de imagem disponíveis (sugeridas + descobertas no Docker Hub)
  ls      Lista as apps Odoo
  create  Cria uma app Odoo
  get     Detalhe de uma app (overview, módulos, backups)
  ops     Operações odoo-bin: restart|upgrade-all|update-list
  module  Módulos
  backup  Dispara um backup (.dlxb)
  files   Lista os ficheiros editáveis dos módulos
  file    Ficheiros de módulos
  help    Print this message or the help of the given subcommand(s)

Options:
      --api <API>      URL da API (ex.: https://host:9443). Também `DELONIX_API`
      --insecure       Aceita certificados TLS self-signed (a API vem assim por omissão). Também `DELONIX_API_INSECURE=1`
      --token <TOKEN>  Token Bearer explícito (sobrepõe `DELONIX_TOKEN`/ficheiro de login)
      --json           Imprime a resposta em JSON em vez de tabela legível, onde aplicável
  -h, --help           Print help

Exemplos

Criar
delonixctl odoo create a-minha-erp --version 17 --domain erp.example.com
Instalar um módulo
delonixctl odoo module install a-minha-erp o_modulo --upgrade

Exemplos

Aplicar

De stdin
