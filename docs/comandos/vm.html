delonix vm

microVMs declarativas: create, ls, status, stop, rm, apply.

MicroVMs geridas pelo trait VmBackend — Cloud Hypervisor ou libvirt. O create é idempotente (cria ou auto-recupera) e suporta cloud-init por instância: --hostname, --ssh-key e --user-data geram um ISO NoCloud automaticamente. É a camada que o delonix cluster kubeadm usa para provisionar nós.

microVMs declarativas: create/ls/stop/rm/status

Usage: delonix vm <COMMAND>

Commands:
  create  Cria (ou auto-recupera) uma VM
  ls      Lista as VMs
  status  Estado actual (reconcilia liveness/IP com o backend)
  stop    Pára a VM (preserva disco/registo)
  rm      Remove a VM (pára + apaga overlay/estado)
  apply   Aplica os documentos `kind: Vm` de um manifesto (`delonix_vm::create` já é idempotente por nome — cria ou auto-recupera)
  help    Print this message or the help of the given subcommand(s)

Options:
  -h, --help  Print help

vm create

Cria (ou auto-recupera) uma VM

Usage: delonix vm create [OPTIONS] --disk <DISK> <NAME>

Arguments:
  <NAME>  

Options:
      --disk <DISK>
          Disco base (qcow2/raw) — vira overlay por-VM
      --vcpus <VCPUS>
          [default: 1]
      --memory <MEMORY>
          Memória (`"2G"`/`"1024M"`) [default: 1G]
      --network <NETWORK>
          Rede do ingress para o tap (`delonix network create` primeiro) [default: bridge]
      --kernel <KERNEL>
          Kernel para direct boot
      --initrd <INITRD>
          
      --firmware <FIRMWARE>
          Firmware, alternativa ao kernel (cloud images)
      --cmdline <CMDLINE>
          
      --seed <SEED>
          ISO cloud-init (NoCloud) já pronta — se dado, tem prioridade sobre `--hostname`/`--ssh-key`/`--user-data` (esses geram a ISO; este usa-a directamente)
      --hostname <HOSTNAME>
          Hostname a aplicar no primeiro boot (gera a ISO NoCloud se nenhum `--seed` explícito for dado)
      --ssh-key <SSH_KEYS>
          Chave SSH pública autorizada, `ssh-ed25519 AAAA...` ou `@caminho` para ler de um ficheiro. Repetível
      --user-data <USER_DATA>
          `user-data` cloud-init próprio (substitui totalmente o gerado por omissão) — controlo completo para quem precisar
      --restart-policy <RESTART_POLICY>
          `no`|`on-failure`|`always`
      --hugepages
          
      --cpu-affinity <CPU_AFFINITY>
          Afinidade de cores, ex.: `8-15`
      --device <DEVICES>
          VFIO PCI passthrough, repetível
      --backend <BACKEND>
          `cloud-hypervisor`|`libvirt` (omitir = auto-deteção)
      --net-mode <NET_MODE>
          Só libvirt: `user`|`nat`|`bridge`
      --bridge <BRIDGE>
          Nome da bridge (net-mode=bridge) ou rede libvirt (nat)
  -h, --help
          Print help

VM a partir da imagem dourada, com chave SSH
