delonix system

O motor em si: events, info, df, prune, monitor, thermal.

Introspecção e manutenção. system prune é o GC (recupera espaço: containers parados, dirs órfãos, imagens dangling, blobs CAS, hostfwds órfãos, redes vazias); system df mostra o uso de disco; system monitor segue ligações/conntrack; system events o fluxo de eventos.

The engine itself: events, state and disk usage

Usage: delonix-x86_64-v3-linux system [OPTIONS] <COMMAND>

Commands:
  events   Eventos do motor (create/start/die/remove/…), do mais antigo ao mais recente. Sem daemon, o registo é um log append-only partilhado — cada comando acrescenta a sua linha (ver `delonix_runtime_core::events`)
  info     Estado do motor: rootless?, delegação de cgroup, infra de rede, contagens
  df       Uso de disco por área (imagens, containers, volumes, imagens VM)
  virt     Virtualização do host: hipervisor, KVM, virtio — e o que há a afinar
  prune    Recupera espaço: remove containers parados, imagens sem uso, blobs do CAS que ninguém referencia, cgroups vazios e — o que mais espaço liberta — **directórios de containers órfãos** (de nós/containers que morreram abruptamente sem `rm`, sem entrada no registo)
  monitor  Ligações de rede activas por container (via conntrack): quem entra, quem sai, e entre containers. Actualiza em contínuo (ver `--no-stream`)
  thermal  Governador térmico: baixa o orçamento de CPU do Delonix quando o CPU aquece e repõe-no quando arrefece. Corre em contínuo (ver `--once`)
  help     Print this message or the help of the given subcommand(s)

Options:
      --l18n <en|pt>  Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
  -h, --help          Print help
