delonix build

Constrói uma imagem a partir de um Dockerfile ou Delonixfile.

Build sem daemon nem BuildKit: sobe um container de trabalho, corre cada RUN por exec, aplica COPY no rootfs (confinado ao contexto — path traversal é rejeitado) e empacota o resultado. Sem -f, procura primeiro um Delonixfile no contexto e só depois um Dockerfile — a gramática é a mesma, com extensões (SCAN, CPUS, MEMORY, SECURITY, HEALTHCHECK). Limitação conhecida: só single-stage (multi-stage é recusado com erro claro).

Constrói uma imagem a partir de um Dockerfile

Usage: delonix build [OPTIONS] --tag <TAG> [CONTEXT]

Arguments:
  [CONTEXT]  Contexto de build (default: `.`) — raiz para `COPY` [default: .]

Options:
  -f, --file <FILE>  Caminho do Dockerfile (default: `<contexto>/Dockerfile`)
  -t, --tag <TAG>    Tag da imagem resultante (`repo:tag`)
  -h, --help         Print help

Exemplos

Build com tag
delonix build -t minha-app:1.0 .
Delonixfile explícito
delonix build -t api:dev -f Delonixfile ./servico
