delonix tunnel

Expõe uma porta local à internet pública via pinggy/ngrok/cloudflare (`kind: Tunnel`).

Faz UMA coisa: leva tráfego da internet pública até UMA porta local — sem conta, sem IP público, sem configurar o router. Junta-se ao httproute apontando --local-port para a porta onde o proxy L7 escuta, e o routing por Host do lado de lá continua a decidir para que container vai cada pedido — uma só URL pública, vários backends. Três providers, cada um o binário/mecanismo REAL desse serviço (nunca simulado): pinggy (zero binário extra — ssh puro, já uma dependência do projecto), ngrok (precisa do agente ngrok no PATH; a URL pública sai da API local do próprio agente) e cloudflare (precisa de cloudflared; por agora só o quick-tunnel efémero *.trycloudflare.com, sem conta — um tunnel NOMEADO com domínio próprio precisa da API do Cloudflare, ainda por implementar).

Expose a local port to the public internet via pinggy/ngrok/cloudflare (`kind: Tunnel`) — pair with `httproute`'s listening port to route by Host header behind ONE public URL

Usage: delonix-x86_64-linux tunnel [OPTIONS] <COMMAND>

Commands:
  apply     Apply the `kind: Tunnel` documents of a manifest (idempotent)
  expose    One-shot expose of a local port, no manifest needed
  ls        List tunnels (state + public URL)
  describe  Human-readable detail of one tunnel
  rm        Stop and remove a tunnel
  help      Print this message or the help of the given subcommand(s)

tunnel expose

One-shot expose of a local port, no manifest needed

Usage: delonix-x86_64-linux tunnel expose [OPTIONS] --provider <PROVIDER> --local-port <LOCAL_PORT>

Options:
      --l18n <en|pt>                 Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
      --name <NAME>                  Name (default: `tunnel-<port>`)
      --provider <PROVIDER>          `pinggy` | `ngrok` | `cloudflare`
      --local-port <LOCAL_PORT>      
      --hostname <HOSTNAME>          
      --token <TOKEN>                
      --token-secret <TOKEN_SECRET>  
  -h, --help                         Print help

Validado ao vivo nesta mesma sessão: tráfego HTTPS real da internet chegou a um servidor local através do tunnel (HTTP 200) usando exactamente este comando.

Exemplos

Expor uma porta local sem escrever manifesto (pinggy, grátis, efémero)
delonix tunnel expose --name demo --provider pinggy --local-port 8080
tunnel/demo: running — https://oxipg-197-148-40-67.free.pinggy.net
