delonix volumes

Volumes nomeados e bind mounts: create, ls, inspect, rm, apply.

Wrapper fino sobre o VolumeStore. No container run, -v nome:/destino[:ro] resolve para um volume nomeado (criado on-demand) e -v /host:/destino[:ro] para um bind mount — a distinção é automática.

Volumes nomeados: create/ls/rm/inspect

Usage: delonix volumes <COMMAND>

Commands:
  create   Cria um volume nomeado
  ls       Lista os volumes
  inspect  Detalhe de um volume (inclui uso real em disco)
  rm       Remove um volume
  apply    Aplica os documentos `kind: Volume` de um manifesto (idempotente por nome)
  help     Print this message or the help of the given subcommand(s)

Options:
  -h, --help  Print help

volumes create

Cria um volume nomeado

Usage: delonix volumes create [OPTIONS] <NAME>

Arguments:
  <NAME>  

Options:
      --driver <DRIVER>    `local` (default) ou `nfs` [default: local]
      --device <DEVICE>    Dispositivo/export (driver `nfs`)
      --options <OPTIONS>  Opções de montagem adicionais (driver `nfs`)
      --quota <QUOTA>      Quota (ex.: `2g`) — só é aplicada se `--quota` for dado
  -h, --help               Print help

Exemplos

Com quota e driver nfs disponíveis
delonix volumes create dados --quota 10G
