delonix ingress

Firewall de ENTRADA (regras L4 + publishes DNAT) de um container na SDN.

Metade da superfície unificada de firewall (a outra é egress). Edita a única fonte de verdade — o ContainerFw por container, aplicado como regras nft na chain de ingress. ingress governa a ENTRADA: regras allow/deny por [proto/]porta e CIDR, a política por omissão, e os publishes DNAT. Só actua em containers numa rede custom (têm IP na delonix0); --net host é recusado.

INBOUND firewall (L4 rules + DNAT publishes) for a container on the SDN

Usage: delonix-x86_64-linux ingress [OPTIONS] <COMMAND>

Commands:
  allow      Allow inbound traffic to a container: `[proto/]port` from an optional CIDR
  deny       Deny inbound traffic to a container (same shape as `allow`)
  policy     Set the default inbound policy when no rule matches
  publish    Publish a host port to the container (DNAT through the ingress)
  unpublish  Remove a published host port
  ls         Show the inbound firewall (policy + rules) and published ports
  rm         Remove inbound rule(s) matching `[proto/]port` (all protos if none given)
  clear      Remove all inbound rules (keeps published ports)
  help       Print this message or the help of the given subcommand(s)

Options:
      --l18n <en|pt>  Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
  -h, --help          Print help

ingress allow

Allow inbound traffic to a container: `[proto/]port` from an optional CIDR

Usage: delonix-x86_64-linux ingress allow [OPTIONS] <CONTAINER> <PORT>

Arguments:
  <CONTAINER>  
  <PORT>       `tcp/5432`, `udp/53`, `5432` (any proto), or `tcp/*` (all ports)

Options:
      --from <FROM>   Only from this source CIDR (default: anywhere)
      --l18n <en|pt>  Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
      --note <NOTE>   Free-form note kept with the rule
  -h, --help          Print help

Exemplos

Deixar entrar Postgres só da própria SDN
delonix ingress allow db tcp/5432 --from 10.219.0.0/16

Usage: delonix-x86_64-linux ingress deny [OPTIONS] <CONTAINER> <PORT>

Arguments:
  <CONTAINER>  
  <PORT>

      --from <FROM>

Exemplos

Bloquear uma porta específica

ingress policy

Set the default inbound policy when no rule matches

Usage: delonix-x86_64-linux ingress policy [OPTIONS] <CONTAINER> <POLICY>

Arguments:
  <CONTAINER>  
  <POLICY>     [possible values: allow, deny]

Options:
      --l18n <en|pt>  Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
  -h, --help          Print help

Exemplos

Default-deny (allowlist)

ingress publish

Publish a host port to the container (DNAT through the ingress)

Usage: delonix-x86_64-linux ingress publish [OPTIONS] <CONTAINER> <SPEC>

Arguments:
  <CONTAINER>  
  <SPEC>       `hostPort:containerPort[/tcp|udp]` or just `port`

Options:
      --l18n <en|pt>  Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
  -h, --help          Print help

Exemplos

Publicar uma porta pelo ingress (DNAT)
delonix ingress publish web 8080:80

Options:
      --l18n <en|pt>  Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
  -h, --help          Print help

Exemplos

Ver regras + publishes
