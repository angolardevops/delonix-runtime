delonix storage

Volumes de REDE (NFS/CIFS/WebDAV) montáveis, estilo PersistentVolume do k8s.

Monta pastas de um NAS (TrueNAS/Synology/Samba/Nextcloud) como volumes nomeados. Por baixo é um volume do delonix-volume com driver de rede — mount -t nfs|cifs|davfs. A password vem do cofre (--password-secret), nunca do argv. Ligado ao stack apply (ordem Network→Volume→Storage→Image→Vm→Container). Montar precisa de CAP_SYS_ADMIN.

Armazenamento de REDE (NFS/CIFS/WebDAV) montável como volume — estilo PV do k8s

Usage: delonix storage [OPTIONS] <COMMAND>

Commands:
  create   Cria (e monta) um armazenamento de rede
  ls       Lista os armazenamentos de rede (volumes com driver de rede)
  inspect  Detalhe de um armazenamento
  rm       Remove (e desmonta) um armazenamento. Os DADOS ficam no NAS — só se desliga a montagem local, como o docker
  apply    Aplica os documentos `kind: Storage` de um manifesto
  help     Print this message or the help of the given subcommand(s)

Options:
      --l18n <en|pt>  Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
  -h, --help          Print help

storage create

Cria (e monta) um armazenamento de rede

Usage: delonix storage create [OPTIONS] --type <TYPE> --server <SERVER> --share <SHARE> <NAME>

Arguments:
  <NAME>  

Options:
      --l18n <en|pt>
          Output language: `en` (default) or `pt` (Portuguese, pt_AO). Also settable via `$DELONIX_L18N`. Global — works before any subcommand
      --type <TYPE>
          Tipo: `nfs` | `cifs`/`smb` (Samba/Windows) | `webdav` (Nextcloud/ownCloud) [possible values: nfs, cifs, smb, webdav]
      --server <SERVER>
          Servidor (host/IP), ou o URL base no caso do `webdav`
      --share <SHARE>
          Export/partilha: caminho NFS (`/mnt/pool/media`), nome da share CIFS (`media`), ou o caminho no URL WebDAV (`/remote.php/dav/...`)
      --username <USERNAME>
          Utilizador (cifs/webdav)
      --password <PASSWORD>
          Password (cifs/webdav) — preferir `--password-secret` para não a expor
      --password-secret <PASSWORD_SECRET>
          Segredo do cofre com a chave `password` (cifs/webdav) — não vaza no histórico
      --read-only
          Monta só-de-leitura
      --options <OPTIONS>
          Opções de mount extra (`vers=4.1,soft`), acrescentadas às derivadas
  -h, --help
          Print help

Exemplos

NFS de um TrueNAS
delonix storage create media --type nfs --server 10.0.0.5 --share /mnt/pool/media
SMB/CIFS com password do cofre
delonix storage create docs --type cifs --server nas --share docs --username user --password-secret nas-pass
