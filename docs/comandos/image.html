delonix image

Imagens OCI: pull, ls, rm, export — e, com --vm, as imagens VM douradas (build/push).

Gestão de imagens de container (registos OCI: Docker Hub, ghcr.io, …) com verificação de digest no pull. Com --vm, o MESMO grupo opera sobre as imagens VM douradas (um .qcow2 + metadados por imagem): Ubuntu cloud image + kubeadm/kubelet/kubectl + delonix-cri — a base do delonix cluster.

Imagens: pull/ls/rm/export. Com `--vm`: imagens VM douradas (ls/pull/push/build) em vez de imagens de container

Usage: delonix image [OPTIONS] <COMMAND>

Commands:
  pull    Puxa uma imagem de um registo
  ls      Lista imagens locais
  rm      Remove uma imagem local
  export  Exporta um bundle OCI runtime (rootfs + config.json) para `runc`/`crun`
  apply   Aplica os documentos `kind: Image` de um manifesto (`pull` idempotente por referência; `build` reconstrói e substitui a tag a cada apply)
  login   Autentica num registo OCI (guarda as credenciais em `<root>/auth.json`, formato docker/podman). A password vem SEMPRE do stdin — nunca de um argumento (ficaria no histórico da shell e no /proc)
  logout  Remove as credenciais guardadas de um registo
  vm      Imagens VM douradas (`<root>/vm-images/`): ls/pull/push/build. Equivalente a `image --vm <cmd>` (forma antiga, mantida)
  push    (só com `--vm`) Publica uma imagem VM local num registo OCI
  build   (só com `--vm`) Constrói a imagem VM dourada (Ubuntu + kubeadm/kubelet/ kubectl + `delonix-cri`)
  help    Print this message or the help of the given subcommand(s)

Options:
      --vm    Opera sobre imagens VM (`<root>/vm-images/`) em vez de imagens de container — activa os subcomandos `push`/`build`
  -h, --help  Print help

image export

Exporta um bundle OCI runtime (rootfs + config.json) para `runc`/`crun`

Usage: delonix image export <IMAGE> <DIR>

Arguments:
  <IMAGE>  
  <DIR>    

Options:
  -h, --help  Print help

Exemplos

Bundle OCI runtime para correr com runc/crun
delonix image export alpine:3.19 /tmp/bundle && sudo runc run -b /tmp/bundle teste

image build

(só com `--vm`) Constrói a imagem VM dourada (Ubuntu + kubeadm/kubelet/ kubectl + `delonix-cri`)

Usage: delonix image build [OPTIONS] --tag <TAG>

Options:
  -t, --tag <TAG>
          
      --ubuntu-release <UBUNTU_RELEASE>
          [default: 26.04]
      --k8s-version <K8S_VERSION>
          
      --extra-package <EXTRA_PACKAGES>
          
      --extra-run <EXTRA_RUN>
          
      --cri-bin <CRI_BIN>
          
      --no-compress
          Não comprimir o qcow2 final (maior, mas sem custo de descompressão nas leituras do backing file em runtime)
      --offline
          Obter os .deb do k8s no HOST (verificados) e instalá-los com `dpkg` — o appliance corre sem rede. Dispensa DHCP/DNS no guest
  -h, --help
          Print help

Exemplos

Construir a imagem VM dourada (descarrega Ubuntu, valida SHA256SUMS, virt-customize)
delonix image --vm build --name k8s-golden --k8s-version 1.34
