Cheatsheet

Todos os grupos de comandos e subcomandos, num só sítio. Gerado do --help real do binário.

Tarefas comuns

Serviço web, sem root, sem daemon
delonix container run -d --name web -p 8080:80 nginx
Shell descartável
delonix container run --rm -it alpine sh
Rede própria + publicar pelo ingress
delonix network create backend
delonix container run -d --net backend -p 8443:443 caddy
Trocar uma porta a QUENTE (sem reiniciar)
delonix container update web --publish-add 9090:80
Firewall: só deixar entrar Postgres da SDN
delonix ingress allow db tcp/5432 --from 10.219.0.0/16
delonix ingress policy db deny
Firewall: egress da rede só p/ DNS + CIDRs
delonix egress net backend allowlist --to 10.0.0.0/8
Tráfego por container ao vivo (eBPF)
sudo delonix flow --watch
Volume de rede de um NAS (NFS)
delonix storage create media --type nfs --server 10.0.0.5 --share /mnt/pool/media
Segredo no cofre (não no argv)
printf 's3nha' | delonix secret create db-pass
microVM com cloud-init
delonix vm create node1 --disk base.qcow2 --ssh-key @~/.ssh/id_ed25519.pub
Cluster Kubernetes do zero
delonix cluster kubeadm --name lab --control-plane 1 --workers 2
Aplicar um manifesto inteiro
delonix stack apply -f delonix-manifest.yaml
Persistir os containers no arranque
delonix boot enable
Recuperar espaço (GC)
delonix system prune

Todos os grupos

delonix container

ComandoO que faz
container dashDashboard (KPIs + tabela + problemas) dos containers — TUI interactivo, ou `--once` para um snapshot de texto
container initInitialize a project with a Delonixfile + manifest — files ALREADY FILLED IN (images included), ready to use without editing anything
container runRun a container from an image (pulls it if missing)
container psList containers [alias: ls]
container start(Re)start stopped/crashed containers, reusing the persistent rootfs (writes made inside the container survive, like in docker) and the same network/ports/volumes as the original `run`. Always detached
container stopStop one or more containers (SIGTERM, then SIGKILL)
container rmRemove one or more containers
container pauseSuspend a container's processes (cgroup v2 freezer) — the state stays in memory, unlike `stop`. Resume with `unpause`
container unpauseResume a container suspended with `pause`
container commitCreate an image from a container's CURRENT rootfs state (whatever was written inside becomes a new layer)
container sshInteractive shell inside a container (shortcut for `exec -t`): with no command, it tries `bash` and falls back to `sh`, which exists in any image
container healthcheckRun the image's `HEALTHCHECK` inside the container. Exits with 1 if `unhealthy` — usable in a script/CI
container topProcesses running inside a container (read from `cgroup.procs`)
container diffFiles changed relative to the image: `A` = created/changed, `D` = deleted
container cpCopy files between the host and a container. Exactly one side is `container:/path` (e.g. `delonix container cp web:/etc/nginx.conf .`)
container execExecute a command inside a running container
container inspectShow the full spec of one or more containers (Store JSON)
container describeHuman-readable detail of one or more containers, `kubectl describe`-style (for humans; use `inspect` for script-consumable JSON)
container update**Reconfigure a RUNNING container without stopping it** — ports, volumes, networks, and bandwidth cap
container statsResource usage (CPU/memory/PIDs) of the running containers — one sample and exits (no stream). With no IDs, shows all running ones
container logsShow the logs (detached containers)
container applyApply the `kind: Container` documents of a manifest (idempotent by name — an existing container with that name is neither recreated nor checked for spec drift, see `cmd::manifest`)

delonix image

ComandoO que faz
image dashDashboard (KPIs + tabela) das imagens — TUI interactivo, ou `--once` snapshot
image pullPuxa uma imagem de um registo
image lsLista imagens locais
image describeDetalhe legível de uma ou mais imagens, ao estilo `kubectl describe` (tags/digest/tamanho/layers + o config OCI: entrypoint/cmd/env/workdir). Com `--vm`, descreve imagens VM douradas
image tagDá outro nome/tag a uma imagem local (não copia nada — é só um nome novo para o mesmo conteúdo)
image historyLayers de uma imagem (digest + tamanho), da base para o topo
image verifyVerifica a assinatura cosign de uma imagem local contra uma chave pública
image scanSBOM + scan de CVE de uma imagem (lê os layers do CAS, sem correr nada). Puxa a imagem se faltar. Ver `--sbom`, `--fail-on`, `--update`
image rmRemove uma imagem local
image exportExporta um bundle OCI runtime (rootfs + config.json) para `runc`/`crun`
image applyAplica os documentos `kind: Image` de um manifesto (`pull` idempotente por referência; `build` reconstrói e substitui a tag a cada apply)
image loginAutentica num registo OCI (guarda as credenciais em `<root>/auth.json`, formato docker/podman). A password vem SEMPRE do stdin — nunca de um argumento (ficaria no histórico da shell e no /proc)
image logoutRemove as credenciais guardadas de um registo
image vmImagens VM douradas (`<root>/vm-images/`): ls/pull/push/build. Equivalente a `image --vm <cmd>` (forma antiga, mantida)
image pushPublica uma imagem local num registo OCI. Sem `target`, publica sob a própria referência da imagem. Com `--vm`, o `target` é obrigatório
image build(só com `--vm`) Constrói a imagem VM dourada (Ubuntu + kubeadm/kubelet/ kubectl + `delonix-cri`)

delonix build

Constrói uma imagem a partir de um Dockerfile ou Delonixfile.

ComandoO que faz
vm dashDashboard (KPIs + tabela) das VMs — TUI interactivo, ou `--once` snapshot

vm describeDetalhe legível de uma ou mais VMs, ao estilo `kubectl describe` (para humanos; use `status` para a vista compacta de sempre). Inclui o estado AO VIVO — `delonix_vm::status` reconcilia liveness/IP com o backend
vm stopPára a VM (preserva disco/registo)
vm rmRemove a VM (pára + apaga overlay/estado)
vm applyAplica os documentos `kind: Vm` de um manifesto (`delonix_vm::create` já é idempotente por nome — cria ou auto-recupera)

delonix stack

ComandoO que faz
stack initInicializa um projecto COMPLETO: Delonixfile + manifesto + cluster + README — ficheiros JÁ PREENCHIDOS (imagens incluídas), prontos a usar sem editar nada
stack applyAplica todos os Kinds do manifesto (Network → Volume → Image → Vm → Container)
stack lsDetalhe do stack ao estilo `kubectl describe`: cada recurso DECLARADO no manifesto e se está, ou não, presente na máquina
stack describe
stack validateValida o manifesto SEM tocar em nada (dry-run): resolve as referências cruzadas (`Container.network`/`.volumes`, `Vm.network`, `Ingress/Egress. target`) contra o que o manifesto declara MAIS o que já existe nos stores. Sai com erro se alguma referência ficar por resolver — é a rede de segurança contra um `apply` que só falharia a meio (fail-fast, sem rollback)

delonix cluster

ComandoO que faz
cluster initInicializa um projecto com os manifestos de cluster (kind/vm/ssh) — ficheiros JÁ PREENCHIDOS (imagens incluídas), prontos a usar sem editar nada
cluster createCria um cluster Kubernetes local **sem manifesto e sem Docker** (modo kind nativo): arranca os nós `kindest/node` no próprio motor Delonix e faz o bootstrap com `kubeadm`. Sem flags = 1 control-plane pronto a usar
cluster lsLista os clusters e o estado dos seus nós [alias: list]
cluster deleteRemove um cluster do modo kind (pára e apaga os nós + kubeconfig)
cluster applyAplica o(s) documento(s) `kind: Cluster` de um manifesto
cluster kubeadmProvisiona VMs (imagem VM dourada) + bootstrap `kubeadm` — do zero a um cluster a funcionar, sem escrever um manifesto à mão

delonix secret

secret lsLista os segredos (nome + nº de chaves; valores NUNCA mostrados)

secret applyAplica os documentos `kind: Secret` de um manifesto (declarativo — cria o segredo sem precisar de `secret create` na CLI)

delonix ingress

ComandoO que faz
ingress allowAllow inbound traffic to a container: `[proto/]port` from an optional CIDR
ingress denyDeny inbound traffic to a container (same shape as `allow`)
ingress policySet the default inbound policy when no rule matches
ingress publishPublish a host port to the container (DNAT through the ingress)
ingress unpublishRemove a published host port
ingress lsShow the inbound firewall (policy + rules) and published ports
ingress clearRemove all inbound rules (keeps published ports)

delonix egress

ComandoO que faz
egress allowAllow outbound traffic from a container: `[proto/]port` to an optional CIDR
egress denyDeny outbound traffic from a container (same shape as `allow`)
egress policySet the default outbound policy when no rule matches
egress netGovern a whole network's egress to the Internet
egress hostAllow a network's egress to a HOSTNAME (and `*.hostname`), learnt live from DNS answers — the FQDN allowlist nft/CIDR can't express. Repeatable
egress lsShow the outbound firewall (policy + rules)
egress showShow a NETWORK's egress policy: CIDR allowlist, FQDN hosts, and the IPs currently learnt from DNS for those hosts
egress clearRemove all outbound rules

delonix flow

Tráfego por-container ao vivo — datapath eBPF (degrada para contadores veth).

delonix boot

ComandoO que faz
boot enableInstall + enable systemd units for the RUNNING containers, so they come back up when the host boots. Rootless uses user units + linger
boot disableDisable + remove the generated boot units
boot statusShow boot-persistence status (installed units + mode)

delonix system

ComandoO que faz
system eventsEventos do motor (create/start/die/remove/…), do mais antigo ao mais recente. Sem daemon, o registo é um log append-only partilhado — cada comando acrescenta a sua linha (ver `delonix_runtime_core::events`)
system infoEstado do motor: rootless?, delegação de cgroup, infra de rede, contagens
system dfUso de disco por área (imagens, containers, volumes, imagens VM)
system virtVirtualização do host: hipervisor, KVM, virtio — e o que há a afinar
system pruneRecupera espaço: remove containers parados, imagens sem uso, blobs do CAS que ninguém referencia, cgroups vazios e — o que mais espaço liberta — **directórios de containers órfãos** (de nós/containers que morreram abruptamente sem `rm`, sem entrada no registo)
system monitorLigações de rede activas por container (via conntrack): quem entra, quem sai, e entre containers. Actualiza em contínuo (ver `--no-stream`)
system thermalGovernador térmico: baixa o orçamento de CPU do Delonix quando o CPU aquece e repõe-no quando arrefece. Corre em contínuo (ver `--once`)

delonix kube

ComandoO que faz
kube generateGera um `kind: Pod` a partir de um container (ou de todos os membros de um pod) e imprime-o em stdout

delonix netns

ComandoO que faz
netns upBring the ingress infra up (idempotent): holder netns + delonix0 + single slirp
netns statusShow the ingress infra status (holder/slirp pids, bridge, refcount)
netns downForce tear-down of the ingress infra (kills slirp + holder, frees the netns)
netns attachAttach a netns to delonix0 via veth (the holder is the netns/veth factory)
netns detachDetach (and destroy) a previously attached netns
netns execRun a command inside an attached netns (exercises the runtime join path)
netns publishPublish a port through the ingress (add_hostfwd + DNAT) to a container
netns unpublishUnpublish a host port from the ingress
netns firewallApply (or clear) a container's parameterizable firewall AT THE INGRESS
